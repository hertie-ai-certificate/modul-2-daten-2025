2) Requests: Das Internet aus dem Notebook ansprechen¶

Was ist ein HTTP-Request?¶

Wenn wir eine Webseite im Browser ffnen, passiert technisch Folgendes: Unser Computer schickt eine standardisierte Anfrage (Request) an den Server der Webseite.
Der Server antwortet mit einer Antwort (Response) meist in Form von HTML-Code, Bildern oder anderen Dateien.

Dieses Protokoll heit HTTP (Hypertext Transfer Protocol).
Es ist ein weltweiter Standard, der nichts mit Python zu tun hat jedes Programm, jeder Browser und auch viele Maschinenkommunikationen nutzen HTTP, um bers Internet Daten auszutauschen.

Ein Request besteht typischerweise aus:

  • einer URL (z. B. https://example.com),
  • einer Methode (z. B. GET, POST, PUT beschreibt, was man tun will),
  • und optional Parametern oder Daten, die mitgeschickt werden.

Requests in Python¶

In Python knnen wir solche HTTP-Anfragen mit der Library requests verschicken.
Diese Bibliothek kmmert sich darum, dass alles im richtigen Format bertragen wird und wir die Antwort einfach weiterverarbeiten knnen.

Mit requests.get("https://example.com") wird also exakt dieselbe Anfrage geschickt, wie unser Browser sie stellen wrde nur ohne sichtbare Benutzeroberflche.

requests ist keine Kernkomponente von Python selbst, gehrt aber zu den Standardbibliotheken, die in quasi jeder Version von Python vorinstalliert sind.

3) Eine API abrufen: Wechselkurse¶

Jetzt nutzen wir wieder requests, aber diesmal nicht, um eine Webseite aufzurufen, sondern um eine API anzusprechen.

Eine API (Application Programming Interface) ist eine Schnittstelle, ber die Computerprogramme standardisiert miteinander kommunizieren.
Im Gegensatz zu normalen Webseiten, die fr Menschen gestaltet sind, sind APIs von Anfang an fr Maschinen gemacht.
Sie liefern Daten immer in klar definierten Strukturen (meist JSON oder XML), damit Programme sie automatisch weiterverarbeiten knnen.

Beispiel: Wechselkurse¶

Wir verwenden hier exchangerate.host, eine kostenlose ffentliche API, die aktuelle Whrungswechselkurse bereitstellt.

Das Prinzip ist dabei dasselbe wie beim Webseitenaufruf:
Wir schicken einen standardisierten HTTP-Request (z. B. GET) an eine bestimmte URL, und der Server antwortet mit Daten diesmal jedoch nicht als HTML-Seite, sondern als Datensatz.

API-Authentifizierung und API-Keys¶

Viele APIs bieten wertvolle oder sensible Informationen an (z. B. personenbezogene Daten oder kostenpflichtige Dienste).
Zudem verursacht jeder Abruf Serverkosten. Deshalb sind die meisten APIs nicht frei zugnglich.

  1. Man registriert sich auf der Webseite des Anbieters.
  2. Man erhlt einen API-Schlssel (API key) eine eindeutige Zeichenfolge aus Buchstaben und Zahlen.
  3. Dieser Schlssel wird bei jeder Anfrage mitgeschickt, um sich zu authentifizieren und die Nutzung zuzuordnen.

Warum APIs wichtig sind¶

APIs sind die Schnittstellen der digitalen Welt sie verbinden Systeme, Anwendungen und Datenquellen miteinander.
Fast jedes moderne Softwareprodukt egal ob im ffentlichen oder privaten Sektor bietet heute eine API an, um Daten auszutauschen oder Funktionen bereitzustellen.

Einige der wichtigsten Arten von APIs sind:

  • Daten-APIs: stellen strukturierte Informationen bereit, z. B. aus Datenbanken, Statistiksystemen oder ffentlichen Registern (Beispiel: EU-Open-Data-Portal, Statistisches Bundesamt).
  • System-APIs: ermglichen die Integration von Verwaltungs-, ERP- oder CRM-Systemen wie SAP, Salesforce oder Dokumentenmanagementsystemen.
  • Sprachmodell- und KI-APIs: bieten Zugriff auf generative KI, bersetzung, Textanalyse oder Bilderkennung (Beispiele: OpenAI, Google Cloud AI, Hugging Face).
  • Echtzeit- und Sensordaten-APIs: liefern aktuelle Messwerte, z. B. Wetterdaten, Verkehr, Energieverbrauch oder Luftqualitt (Beispiel: Open-Meteo, Transport APIs).
  • Zahlungs- und Authentifizierungs-APIs: sorgen fr sichere Identifikation und Transaktionen (z. B. OAuth, PayPal, BundID).

Gemeinsam ist all diesen Schnittstellen, dass sie strukturiert, standardisiert und maschinenlesbar sind im Gegensatz zu Webseiten, die fr Menschen gestaltet wurden.
Dadurch knnen Systeme:

  • Daten automatisiert abrufen oder aktualisieren,
  • Informationen zwischen Anwendungen synchronisieren,
  • Prozesse ber Organisationsgrenzen hinweg verbinden.

APIs sind damit die Sprachkanle moderner Informationssysteme sie machen es mglich, dass einzelne digitale Dienste zu einem vernetzten kosystem werden.

5) OCR einer lokalen Datei per API¶

Jetzt beschftigen wir uns mit Texterkennung (OCR) also der automatischen Umwandlung von Text, der in einem Bild oder Scan enthalten ist, in echten, digitalen Text.

OCR steht fr Optical Character Recognition und ist eine grundlegende Technologie, um analoge oder gescannte Dokumente maschinenlesbar zu machen.

Wenn man etwa ein eingescanntes Formular oder einen alten Boardingpass hat, liegt der Text darin nur als Pixelmuster vor fr ein KI-System oder ein Skript ist das zunchst unbrauchbar.

OCR analysiert das Bild, erkennt Buchstaben und Zahlen und erzeugt daraus eine Textversion, die man weiterverarbeiten kann.

Wir nutzen hier OCR.space, einen Online-Dienst, der diese Texterkennung direkt ber API anbietet. Es gibt viele Wege, OCR zu machen, u.A. direkt in Python via Libraries, oder als

Ablauf¶

Wir laden eine Datei von unserem Computer in die Colab-Umgebung hoch. Das kann z. B. ein PDF oder ein eingescanntes Formular sein.

  1. Wir senden die Datei ber requests.post() an die OCR.space-API.
    Diese erkennt den enthaltenen Text automatisch.
  2. Die API gibt das Ergebnis als JSON zurck.
    Wir lesen das Feld ParsedText aus und zeigen den erkannten Text an.

Warum OCR wichtig ist¶

OCR ist ein entscheidender erster Schritt, wenn man vorhandene Dokumente oder Datenbestnde fr moderne KI-Anwendungen nutzbar machen mchte.
In Verwaltungen, Unternehmen oder Archiven liegen viele Informationen in nicht-digitaler oder schlecht strukturierter Form vor etwa als Scan, Fax oder Formular.
LLMs und andere Sprachmodelle knnen diese Inhalte nur verarbeiten, wenn sie als Text vorliegen.

Allerdings endet der Prozess hier noch nicht:
Der Text, den OCR zurckliefert, ist in der Regel nicht strukturiert es handelt sich um reinen Flietext ohne erkennbare Felder, Tabellen oder Metadaten.
Bevor man ihn mit einem Sprachmodell sinnvoll verwenden kann, mssen daher oft noch weitere Schritte folgen:

  • Bereinigung: Entfernen von Artefakten, Zeilenumbrchen oder fehlerhaften Zeichen.
  • Strukturierung: Zuordnung zu Datenfeldern (z. B. Name, Datum, Preis, Referenznummer).
  • Anreicherung: Kombination mit anderen Datenquellen oder Metadaten.

Erst danach lassen sich die Inhalte zuverlssig durchsuchen, auswerten oder in KI-Systeme einspeisen.
OCR ist also das Tor zur Digitalisierung aber erst der Anfang der eigentlichen Datenverarbeitung.

Hinweise:

  • Sprache passend einstellen (language="deu", "eng" oder "eng+deu").
  • Niedrige Scanauflsung scale=True setzen.

Probiere es mit "pass_1.pdf" und den anderen beiden Dateien aus. Probiere es auch gerne mit irgendwelchen anderen PDFs aus, welche datenschutztechnisch unkritisch sind.

7) Hauptpunkte¶

  1. Webscraping ist eine Methode, um Daten aus ffentlichen Webseiten abzugreifen. So kann theoretisch das gesamte Internet als Datenquelle genutzt werden. In der Praxis ist Webscraping oft unverlsslich, die Daten sind am Ende unstrukturiert, und zunehmend blockieren Webanbieter es ganz.

  2. APIs sind Schnittstellen, ber die Programme strukturiert miteinander kommunizieren knnen. Mit Parametern und Auswahl von Routes kann spezifiziert werden, was genau von einem API gewollt ist.

  3. Was bei einem API "hinter den Kulissen" passiert ist uns "egal": wir schicken nur eine Anfrage, und bekommen eine Antwort.
