Einführung in Webscraping und APIs mit Python¶

Ziel: Verstehen, wie Jupyter-Notebooks funktionieren, was Requests und Webscraping sind, und wie man reale APIs abfragt: Wechselkurse, Wetter, und Texterkennung (OCR).

Zellen mit Shift+Enter ausführen. Code kann direkt bearbeitet werden. Ergebnisse erscheinen unter der jeweiligen Zelle.

1) Wie Jupyter-Notebooks funktionieren¶

Ein Jupyter-Notebook ist eine interaktive Arbeitsumgebung, in der wir Text, Code und Ergebnisse direkt miteinander kombinieren können. Es wird häufig in Datenanalyse, Lehre und Forschung verwendet, weil es Dokumentation und Ausführung in einer Oberfläche vereint. Colab, wo wir gerade sind, ist eine Gratis-Plattform von Google, über die man Jupyter-Notebooks online schreiben und ausführen kann, ohne sich um irgendetwas kümmern zu müssen.

  • Textzellen (wie diese) dienen der Erklärung, Beschreibung oder Dokumentation. Sie werden in Markdown geschrieben und können Überschriften, Listen, Tabellen oder Formeln enthalten.
  • Codezellen enthalten Python-Code, der durch Drücken von Shift + Enter ausgeführt wird. Das Ergebnis erscheint direkt darunter.
  • Zellen werden von oben nach unten ausgeführt. Der Ablauf kann aber manuell gesteuert werden, d. h. man kann auch einzelne Zellen zwischendurch starten.
  • Variablen bleiben erhalten, solange der sogenannte Kernel (die laufende Python-Sitzung) aktiv ist.
  • Ein Neustart des Kernels löscht alle gespeicherten Variablen und den aktuellen Zustand – der Code muss dann erneut ausgeführt werden, um alles wiederherzustellen.

So entsteht ein nachvollziehbarer „Rechenbericht“, der Daten, Analysen und Erläuterungen transparent miteinander verbindet.

In diesem Schritt nutzen wir Pandas. Pandas ist eine von vielen tausenden sogenannten Libraries in Python. Libraries sind Codepakete, die bestimmte Funktionen oder Werkzeuge enthalten, damit man sie nicht jedes Mal selbst schreiben muss. Einige davon sind sehr wichtig für die Arbeit mit Python und werden in quasi jedem Projekt genutzt.

2) Requests: Das Internet aus dem Notebook ansprechen¶

In diesem Abschnitt lernen wir, wie unser Notebook mit dem Internet kommuniziert.
Dazu müssen wir verstehen, was ein sogenannter HTTP-Request ist.

Was ist ein HTTP-Request?¶

Wenn wir eine Webseite im Browser öffnen, passiert technisch Folgendes: Unser Computer schickt eine standardisierte Anfrage (Request) an den Server der Webseite.
Der Server antwortet mit einer Antwort (Response) – meist in Form von HTML-Code, Bildern oder anderen Dateien.

Dieses Protokoll heißt HTTP (Hypertext Transfer Protocol).
Es ist ein weltweiter Standard, der nichts mit Python zu tun hat – jedes Programm, jeder Browser und auch viele Maschinenkommunikationen nutzen HTTP, um übers Internet Daten auszutauschen.

Ein Request besteht typischerweise aus:

  • einer URL (z. B. https://example.com),
  • einer Methode (z. B. GET, POST, PUT – beschreibt, was man tun will),
  • und optional Parametern oder Daten, die mitgeschickt werden.

Requests in Python¶

In Python können wir solche HTTP-Anfragen mit der Library requests verschicken.
Diese Bibliothek kümmert sich darum, dass alles im richtigen Format übertragen wird und wir die Antwort einfach weiterverarbeiten können.

💡 Mit requests.get("https://example.com") wird also exakt dieselbe Anfrage geschickt, wie unser Browser sie stellen würde – nur ohne sichtbare Benutzeroberfläche.

requests ist keine Kernkomponente von Python selbst, gehört aber zu den Standardbibliotheken, die in quasi jeder Version von Python vorinstalliert sind.
Importieren muss man sie trotzdem:

import requests

HTML und BeautifulSoup¶

Die Antwort auf einen HTTP-Request ist bei Webseiten meist HTML-Code – also Text, der beschreibt, wie die Seite aufgebaut ist. Damit wir daraus gezielt Informationen (z. B. den Titel) herausfiltern können, verwenden wir die Library BeautifulSoup.

beautifulsoup analysiert HTML-Strukturen und ermöglicht es, gezielt Elemente zu suchen, etwa <title>, <p>, <div> usw. Für unseren Kurs reicht es zu wissen, dass sie „den HTML-Code in eine strukturierte Form bringt, mit der man arbeiten kann“.

Dokumentation: https://pypi.org/project/beautifulsoup4/

    response.raise_for_status()  # Raise an exception for bad status codes

    print("So sieht der Start des HTMLs für diese Seite aus:")

    soup = BeautifulSoup(response.content, 'html.parser')
    h1_tag = soup.find('h1')

        print(f"Kein HTML Tag gefunden")

except requests.exceptions.RequestException as e:
    print(f"Fehler beim Aufruf der URL: {e}")

3) Eine API abrufen: Wechselkurse¶

Jetzt nutzen wir wieder requests, aber diesmal nicht, um eine Webseite aufzurufen, sondern um eine API anzusprechen.

Eine API (Application Programming Interface) ist eine Schnittstelle, über die Computerprogramme standardisiert miteinander kommunizieren.
Im Gegensatz zu normalen Webseiten, die für Menschen gestaltet sind, sind APIs von Anfang an für Maschinen gemacht.
Sie liefern Daten immer in klar definierten Strukturen (meist JSON oder XML), damit Programme sie automatisch weiterverarbeiten können.

Beispiel: Wechselkurse¶

Wir verwenden hier exchangerate.host, eine kostenlose öffentliche API, die aktuelle Währungswechselkurse bereitstellt.

Das Prinzip ist dabei dasselbe wie beim Webseitenaufruf:
Wir schicken einen standardisierten HTTP-Request (z. B. GET) an eine bestimmte URL, und der Server antwortet mit Daten – diesmal jedoch nicht als HTML-Seite, sondern als Datensatz.

API-Authentifizierung und API-Keys¶

Viele APIs bieten wertvolle oder sensible Informationen an (z. B. personenbezogene Daten oder kostenpflichtige Dienste).
Zudem verursacht jeder Abruf Serverkosten. Deshalb sind die meisten APIs nicht frei zugänglich.

Üblich ist folgendes System:

  1. Man registriert sich auf der Webseite des Anbieters.
  2. Man erhält einen API-Schlüssel (API key) – eine eindeutige Zeichenfolge aus Buchstaben und Zahlen.
  3. Dieser Schlüssel wird bei jeder Anfrage mitgeschickt, um sich zu authentifizieren und die Nutzung zuzuordnen.

Beispiel:

params = {"apikey": "MEIN_GEHEIMER_SCHLUESSEL"}
`

So weiß der Anbieter, von wem die Anfrage stammt. Gleichzeitig kann er Zugriffe limitieren oder abrechnen. Ein API-Key sollte nie öffentlich im Code stehen, sondern z. B. in einer Umgebungsvariable gespeichert werden. Was wir hier in diesem Notebook machen ist sehr schlecht 😡 (aber nicht so wild, weil die API Keys alle keine Daten oder Zahlungsmethoden hinterlegt haben).

Warum APIs wichtig sind¶

APIs sind die Schnittstellen der digitalen Welt – sie verbinden Systeme, Anwendungen und Datenquellen miteinander.
Fast jedes moderne Softwareprodukt – egal ob im öffentlichen oder privaten Sektor – bietet heute eine API an, um Daten auszutauschen oder Funktionen bereitzustellen.

Einige der wichtigsten Arten von APIs sind:

  • Daten-APIs: stellen strukturierte Informationen bereit, z. B. aus Datenbanken, Statistiksystemen oder öffentlichen Registern (Beispiel: EU-Open-Data-Portal, Statistisches Bundesamt).
  • System-APIs: ermöglichen die Integration von Verwaltungs-, ERP- oder CRM-Systemen wie SAP, Salesforce oder Dokumentenmanagementsystemen.
  • Sprachmodell- und KI-APIs: bieten Zugriff auf generative KI, Übersetzung, Textanalyse oder Bilderkennung (Beispiele: OpenAI, Google Cloud AI, Hugging Face).
  • Echtzeit- und Sensordaten-APIs: liefern aktuelle Messwerte, z. B. Wetterdaten, Verkehr, Energieverbrauch oder Luftqualität (Beispiel: Open-Meteo, Transport APIs).
  • Zahlungs- und Authentifizierungs-APIs: sorgen für sichere Identifikation und Transaktionen (z. B. OAuth, PayPal, BundID).

Gemeinsam ist all diesen Schnittstellen, dass sie strukturiert, standardisiert und maschinenlesbar sind – im Gegensatz zu Webseiten, die für Menschen gestaltet wurden.
Dadurch können Systeme:

  • Daten automatisiert abrufen oder aktualisieren,
  • Informationen zwischen Anwendungen synchronisieren,
  • Prozesse über Organisationsgrenzen hinweg verbinden.

APIs sind damit die Sprachkanäle moderner Informationssysteme – sie machen es möglich, dass einzelne digitale Dienste zu einem vernetzten Ökosystem werden.

Was passiert technisch?¶

In unserem Code:

  1. Wir schicken einen Request an die URL BASE.
  2. Wir legen einen Timeout (z. B. 20 Sekunden) fest, damit das Programm nicht ewig wartet, falls keine Antwort kommt.
  3. Wir erhalten eine strukturierte Antwort, die wir direkt weiterverarbeiten können.

Der entscheidende Punkt:

💡 API-Antworten folgen immer einer klar definierten Struktur. Dadurch wissen wir genau, welche Felder vorhanden sind und können sie zuverlässig auslesen – anders als beim Webscraping, wo das Seitenlayout sich jederzeit ändern kann.

4) Noch eine API abrufen: Open-Meteo¶

Kein API-Schlüssel nötig. Wir holen stündliche Temperaturdaten für Berlin und erstellen eine einfache Grafik. Hierbei lernen wir noch zwei neue Dinge über APIs:

  • Eine API kann viele verschiedene „Routen“ anbieten – also Endpunkte mit unterschiedlichen URLs, die jeweils andere Daten oder Funktionen bereitstellen. Zum Beispiel nutzen wir hier die Route /forecast, um eine Wettervorhersage zu bekommen. Andere Routen können etwa historische Wetterdaten, Klimadaten oder Windinformationen liefern.

  • Über das Argument params übergeben wir der API Parameter, also zusätzliche Informationen, die steuern, welche Daten wir möchten. Diese Parameter sagen der API in diesem Fall zum Beispiel:

    • Ort: z. B. durch geografische Koordinaten (latitude, longitude)
    • Zeitraum oder Frequenz: z. B. stündlich (hourly) oder täglich (daily)
    • Messgrößen: z. B. Temperatur, Wind, Luftfeuchtigkeit

Die API verarbeitet diese Angaben und liefert die passenden Daten als Antwort zurück – meist im JSON-Format, also einer klar strukturierten Textform, die Python leicht weiterverarbeiten kann.

Welche Routen eine API anbietet und welche Parameter jede Route akzeptiert, steht immer in der Dokumentation. Diese findet man in der Regel auf der Website der API-Anbieter – in diesem Fall hier: https://open-meteo.com/en/docs

👉 Ändere die lat und lon-Werte, um die Vorhersage für eine andere Stadt abzurufen. Schaffst du es, mithilfe der API-Dokumentation was ganz anderes abzurufen, z.B. die Regenwahrscheinlichkeit?

5) OCR einer lokalen Datei per API¶

Jetzt beschäftigen wir uns mit Texterkennung (OCR) – also der automatischen Umwandlung von Text, der in einem Bild oder Scan enthalten ist, in echten, digitalen Text.

OCR steht für Optical Character Recognition und ist eine grundlegende Technologie, um analoge oder gescannte Dokumente maschinenlesbar zu machen.

Wenn man etwa ein eingescanntes Formular oder einen alten Boardingpass hat, liegt der Text darin nur als Pixelmuster vor – für ein KI-System oder ein Skript ist das zunächst unbrauchbar.

OCR analysiert das Bild, erkennt Buchstaben und Zahlen und erzeugt daraus eine Textversion, die man weiterverarbeiten kann.

Wir nutzen hier OCR.space, einen Online-Dienst, der diese Texterkennung direkt über API anbietet. Es gibt viele Wege, OCR zu machen, u.A. direkt in Python via Libraries, oder als

Ablauf¶

👉 Wir laden eine Datei von unserem Computer in die Colab-Umgebung hoch. → Das kann z. B. ein PDF oder ein eingescanntes Formular sein.

  1. Wir senden die Datei über requests.post() an die OCR.space-API.
    Diese erkennt den enthaltenen Text automatisch.
  2. Die API gibt das Ergebnis als JSON zurück.
    Wir lesen das Feld ParsedText aus und zeigen den erkannten Text an.

Warum OCR wichtig ist¶

OCR ist ein entscheidender erster Schritt, wenn man vorhandene Dokumente oder Datenbestände für moderne KI-Anwendungen nutzbar machen möchte.
In Verwaltungen, Unternehmen oder Archiven liegen viele Informationen in nicht-digitaler oder schlecht strukturierter Form vor – etwa als Scan, Fax oder Formular.
LLMs und andere Sprachmodelle können diese Inhalte nur verarbeiten, wenn sie als Text vorliegen.

Allerdings endet der Prozess hier noch nicht:
Der Text, den OCR zurückliefert, ist in der Regel nicht strukturiert – es handelt sich um reinen Fließtext ohne erkennbare Felder, Tabellen oder Metadaten.
Bevor man ihn mit einem Sprachmodell sinnvoll verwenden kann, müssen daher oft noch weitere Schritte folgen:

  • Bereinigung: Entfernen von Artefakten, Zeilenumbrüchen oder fehlerhaften Zeichen.
  • Strukturierung: Zuordnung zu Datenfeldern (z. B. Name, Datum, Preis, Referenznummer).
  • Anreicherung: Kombination mit anderen Datenquellen oder Metadaten.

Erst danach lassen sich die Inhalte zuverlässig durchsuchen, auswerten oder in KI-Systeme einspeisen.
OCR ist also das Tor zur Digitalisierung – aber erst der Anfang der eigentlichen Datenverarbeitung.

  • Sprache passend einstellen (language="deu", "eng" oder "eng+deu").
  • Niedrige Scanauflösung → scale=True setzen.

7) Hauptpunkte¶

  1. Webscraping ist eine Methode, um Daten aus öffentlichen Webseiten abzugreifen. So kann theoretisch das gesamte Internet als Datenquelle genutzt werden. In der Praxis ist Webscraping oft unverlässlich, die Daten sind am Ende unstrukturiert, und zunehmend blockieren Webanbieter es ganz.

  2. APIs sind Schnittstellen, über die Programme strukturiert miteinander kommunizieren können. Mit Parametern und Auswahl von Routes kann spezifiziert werden, was genau von einem API gewollt ist.

  3. Was bei einem API "hinter den Kulissen" passiert ist uns "egal": wir schicken nur eine Anfrage, und bekommen eine Antwort.
